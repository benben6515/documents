# 命名很難

## **Mac 檔案命名速查表**

（為 **終端機搜尋**、**macOS Finder 排序**和 **跨平台相容性** 優化）

---

### **1. 核心規則**

✅ **無空格** — 使用 `-` 或 `_` 取代
✅ **使用零填充數字**（`001`、`002`，…）讓排序自然運作
✅ **堅持使用小寫** 方便輸入且保持一致性
✅ **避免終端機特殊字元**：

```
&  *  ?  (  )  !  $  '  "  [  ]  {  }  <  >  ;  |  \  ~  `
```

✅ **保持名稱簡短** 但描述清楚（20–40 字元為佳）

---

### **2. 推薦格式**

```
NNN-keywords-here.md
```

- **NNN** = 用於排序的零填充數字
- **keywords-here** = 用連字號分隔的小寫單字
- 範例：

```
001-markdown-template.md
002-css-style-guide.md
003-python-scripts.md
```

---

### **3. 為什麼連字號是最佳選擇**

- 終端機不需要跳脫字元
- 在 URL 中運作良好
- 比 `_underscore_style` 看起來更乾淨
- Finder 和 `ls` 排序完全符合預期

---

### **4. 實用終端機技巧**

| 動作                    | 指令               | 範例                              |
| ------------------------- | --------------------- | ------------------------------------ |
| 列出所有檔案            | `ls`                  | `ls`                                 |
| 列出特定數字範圍        | `ls 00[1-3]*`         | 列出 `001` 到 `003`                 |
| 關鍵字搜尋              | `ls *markdown*`       | 找出所有包含「markdown」的檔案      |
| 模糊搜尋                | `fzf`                 | 互動式搜尋（需安裝）                 |
| 在預設應用程式中開啟    | `open file.md`        | 在預設編輯器中開啟                  |
| 批次重新命名            | `rename` 或 `mv` 迴圈 | 依模式重新命名所有檔案              |

---

### **5. 可選的分類前綴**

用於超快過濾：

```
M01-markdown-template.md   # markdown
C01-css-style-guide.md     # css
P01-python-scripts.md      # python
```

然後你可以：

```
ls M*
```

只列出 Markdown 文件。

---

### **6. 範例工作流程**

```bash
touch 001-markdown-template.md
touch 002-css-style-guide.md
touch 003-python-scripts.md

ls 00[1-2]*     # 只顯示 001–002 的檔案
ls *template*   # 任何包含 'template' 的檔案
```

## **Mac 資料夾命名速查表**

對於 **資料夾命名**，原則幾乎與檔案相同，但由於資料夾常用於分組和長期組織，需額外考慮幾點。

這是你的 **Mac 資料夾命名速查表**（終端機友善）。

---

### **1. 資料夾核心規則**

✅ **無空格** — 與檔案相同，堅持使用 `-` 或 `_`
✅ **有序序列的零填充數字**：`001-project-a`、`002-project-b`
✅ **小寫** 以保持一致性和方便輸入
✅ **避免終端機特殊字元**（與檔案相同的禁用清單）
✅ **要有描述性** — 資料夾常在數月後搜尋，避免使用模糊名稱如 `misc` 或 `test`

---

### **2. 推薦格式**

#### **A. 簡單編號 + 關鍵字**

```
001-photos-2025
002-client-reports
003-marketing-assets
```

#### **B. 分類前綴 + 編號 + 關鍵字**

當你有混合類型的資料夾時很有用：

```
DOC-001-annual-reports
IMG-002-event-photos
DEV-003-javascript-scripts
```

- 你可以用 `ls DOC*` 只看文件。

#### **C. 日期前綴（YYYY-MM-DD 或 YYYY-MM）**

非常適合按時間排序的專案：

```
2025-08-13-meeting-notes
2025-08-client-onboarding
```

- 在 Finder 和終端機中完美排序
- `ls 2025-08*` 顯示所有 2025 年 8 月的資料夾

---

### **3. 終端機搜尋技巧**

| 動作            | 指令                            | 範例                     |
| ----------------- | ---------------------------------- | --------------------------- |
| 只列出資料夾     | `ls -d */`                         | 只顯示資料夾名稱           |
| 關鍵字搜尋       | `ls -d *report*/`                  | 包含「report」的資料夾      |
| 範圍搜尋         | `ls -d 00[1-5]*/`                  | 資料夾 001–005             |
| 快速瀏覽         | `cd 002-client-reports`            | 進入資料夾                 |
| 模糊尋找資料夾   | `fd . -t d`（需安裝 `fd`）         | 互動式搜尋                 |

---

### **4. macOS 特殊技巧**

- 在 Finder 中，你可以 **顏色標記** 資料夾（例如：紅色代表緊急，藍色代表存檔），而不會改變名稱 — 與命名規則搭配良好
- 如果你使用 **Spotlight**（`cmd + space`），你一致的資料夾名稱會變成可即時搜尋

---

### **5. 好的 vs 壞的範例**

**好的**

```
001-project-alpha
002-project-beta
2025-01-tax-documents
IMG-005-family-photos
```

**壞的**

```
project 1     # 空格會造成引號困擾
final!!!      # 特殊字元讓終端機混亂
backup-old    # 日期或版本不清楚
```
