## 什麼是規格書？

把規格書想像成**樂高積木**。

在開始建造之前，你需要：

- 你要做什么？（汽車？房子？）
- 你需要哪些零件？
- 零件如何組合在一起？
- 完成後會是什麼樣子？

規格書就是那份計畫。寫下來。在你寫程式碼之前。

**簡單範例：**

```
不好：「我要做一個登入系統」

很好：「使用者可以輸入 email 和密碼。
      如果對了，他們就能登入。
      如果錯了，他們再試一次。
      系統會記住他們是誰。」
```

---

## 為什麼要寫？

### 不會忘記事情

當你把事情寫下來，你不會忘記重要的東西。

**沒有規格書：**「糟糕，我忘記加登出功能了！」

**有規格書：**「就在第 3 頁！」

### 每個人都同意

當每個人都看到計畫，他們都知道要做什么。

- 你知道要建什麼
- 你朋友知道要建什麼
- 同樣的東西！

### 更少錯誤

寫下來幫助你先思考。

你在寫程式碼**之前**就發現問題。

這很好。省時間。

## 7 個部分

好的規格書有 7 個部分。簡單的部分。

### 第 1 部分：這是什麼？

**回答這些問題：**

- 這個東西叫什麼？
- 誰在寫？
- 什麼時候寫的？

**範例：**

```
名稱：使用者登入系統
作者：Sarah
日期：2025 年 1 月 15 日
```

**為什麼重要：**
每個人都知道這是關於什麼的。之後容易找到。

---

### 第 2 部分：我們在解決什麼問題？

**簡單解釋：**

- 現在有什麼問題？
- 為什麼要修它？

**範例：**

```
問題：
使用者無法登入我們的 App。
他們會生氣並離開。
這讓我們賺不到錢。
```

**為什麼重要：**
每個人都明白**為什麼**我們要做這個。

---

### 第 3 部分：我們如何修它？

**這是主要部分。解釋：**

**如何運作：**

```
解決方案：
1. 使用者輸入 email 和密碼
2. 系統檢查是否正確
3. 如果正確 → 使用者登入
4. 如果錯誤 →「再試一次」訊息
```

**當出問題時會發生什麼：**

```
密碼錯誤：
顯示「密碼錯誤」
讓他們試 3 次

沒有 Email：
顯示「找不到 Email」
顯示「註冊」按鈕
```

**為什麼重要：**
這是你的**計畫**。你建這個。

---

### 第 4 部分：還要想什麼？

**要問的問題：**

**這會打破別的東西嗎？**

```
不要打破「忘記密碼」功能。
它仍然需要運作。
```

**它安全嗎？**

```
密碼必須是秘密的。
不要給任何人看。
```

**它會很慢嗎？**

```
應該很快。
少於 1 秒鐘登入。
```

**為什麼重要：**
你在問題發生**之前**就想到它們。

---

### 第 5 部分：我們怎麼知道成功了？

**選擇簡單的檢查：**

**範例：**

```
成功檢查：
✓ 100 個使用者可以登入
✓ 錯誤密碼不會讓他們進入
✓ 少於 1 秒鐘
✓ 在手機和電腦上都能用
```

**為什麼重要：**
完成的時候，你知道它是好的！

---

### 第 6 部分：什麼時候會完成？

**分成小步驟：**

**範例：**

```
時間表：

第 1 週：做登入畫面
第 2 週：連接資料庫
第 3 週：和 10 個人測試
第 4 週：修問題
第 5 週：每個人都能用了！
```

**為什麼重要：**
每個人都知道事情什麼時候發生。沒有驚喜。

---

### 第 7 部分：我們還不知道什麼？

**不知道所有事情沒關係！**

**寫下問題：**

```
問題：
- 如果使用者忘記密碼怎麼辦？
  → 我們下個月加「忘記密碼」
- 他們應該永遠保持登入嗎？
  → 問老闆
- 如果 1000 個人同時試怎麼辦？
  → 測試這個看看
```

**為什麼重要：**
你展現你想過它。其他人幫忙回答。

## 簡單寫作規則

### 用小字詞

**不好：**「實作認證協定」

**很好：**「讓人們登入」

### 短句子

**不好：**「系統將透過安全機制驗證使用者憑證」

**很好：**「使用者登入。很安全。」

### 真實範例

用人們知道的東西：

- 像房子鑰匙
- 像商店收據
- 像電玩關卡

### 簡單解釋專有名詞

**不好：**「協調架構 | Orchestrate the architecture」（那是什麼意思？）

**很好：**「把東西組合在一起 | Put the pieces together」

## 常見錯誤

### 字詞太大

**不好：**「實作認證」
**很好：**「讓人們登入」

**不好：**「部署功能」
**很好：**「放在網站上」

### 忘記什麼

**不好：**「使用者可以登入」
**很好：**「使用者可以登入和登出」

### 不清楚

**不好：**「修登入」
**很好：**「使用者輸入 email 和密碼。如果對，他們登入。」

---

## 快速檢查清單

在你分享規格書之前，檢查：

✓ 我有說什麼是問題嗎？
✓ 我有說如何修它嗎？
✓ 我的字詞小又簡單嗎？
✓ 我有想到如果壞了會怎樣嗎？
✓ 我知道什麼時候會完成嗎？
✓ 我有寫下我還不知道什麼嗎？
